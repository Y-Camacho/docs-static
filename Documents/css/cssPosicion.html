El posicionamiento

Las propiedades FLOAT y POSITION nos permiten modificar la posición natural de cualquier elemento del documento HTML. El navegador posiciona cada elemento teniendo en cuenta el orden en que aparece en el documento (de izquierda a derecha y de arriba a abajo) y el tipo de visualización dependiendo si es un elemento en línea o en bloque. Con FLOAT y POSITION podemos modificar este comportamiento.

Tipos de posicionamiento

  • static (default)
  • relative
  • absolute
  • fixed
  • sticky

Para mover y posicionar los elementos (con relative, absolute, fixed o sticky) utilizaremos las propiedades:

  • top (vertical, eje y)
  • bottom (vertical, eje y)
  • left (horizontal, eje x)
  • right (horizontal, eje x)
  • z-index (profundidad, eje z)

Estas 5 propiedades NO funcionan con el valor por defecto static! Porque static significa respetar el flujo original del posicionamiento de los elementos.

Las propiedades top y left tienen prioridad respecto a bottom y right respectivamente. En caso de conflicto entre aplicar top o bottom, se aplicará top. Lo mismo pasa con left y right.

Posicionamiento static

En este posicionamiento no se aplican left, right, top, bottom ni z-index.

Posicionamiento relative

En este posicionamiento el elemento conserva las dimensiones y el posicionamiento original dentro del flujo de documento. Si se mueve toma como referencia su posición original.

Posicionamiento absolute

El elemento pierde sus dimensiones (width y height) y posición original en el flujo del documento. Si se mueve, puede tomar como referencia dos elementos:

  • El primer ancestro con posicionamiento relativo.
  • Si no encuentra ningún ancestro relativo, se mueve respecto al documento HTML.

Posicionamiento FIXED

Pierde las dimensiones y posición original del flujo del documento. Si se mueve, toma como referencia el documento HTML y queda fija en la posición donde se ha movido. Cuando se haga un scroll, el elemento no se moverá, se quedará fijo.

Posicionamiento Sticky o pegajoso

Es una combinación de posicionamiento relative y fixed. En este tipo de posicionamiento las propiedades top, left, bottom y right no mueven el elemento. En vez de desplazarse, sirven como punto de referencia mientras el elemento no llega a los valores establecidos. Mientras no llegue a las posiciones establecidas por las propiedades, el elemento se comporta relativamente y cuando llega a estas posiciones, queda fijo.

Hay dos opciones para implementarlo:

  • El elemento sticky tiene que ser hijo directo del body.
  • Su elemento contenedor debe tener dimensiones definidas y solamente será sticky dentro de este contenedor.
